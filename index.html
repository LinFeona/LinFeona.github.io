libuv session内存池的设计

可以阅读 更多文章!

session 原理

  • 每个socket 都会对应一个session跟客户端socket 所有的交流都在session中处理

  • session 会提供几个主流的接口
    发送数据 关闭连接 以及获取信息

  • 每当有一个用户进来的时候 就通过session.create 来创建一个session来对接新的用户

  • 每当有一个用户离开的时候 就通过 session.destroy 进行回收

  • session.create 以及 session.destroy 为从内存池中获取以及回收到内存池中

内存池设计原理

  • 一次性创建大量session
  • session.create时 获取一个
  • session.destroy 时 清空数据回收到池中
  • 分配器的管理

多个协议session 原理

  • 若有多个协议的连接 可能session中的数据结构会有所不同
  • session 可以采用继承模式 session基类则提供基础接口子类则区分这些不同的结构
    子类则区分这些不同的结构
  • 上层使用时只管使用session即可

Libuv c语言中的环境配置

可以阅读 更多文章!

简介

  • libuv是跨平台的异步事件库

libuv 下载

  • libuv官网
  • 下载直通车
  • 选择一个版本 下载 gz 后缀的源码

环境配置

  • 项目目录 src win32 3rd 目录 代码 运行时目录 第三方代码

  • 将 libuv/src 与 include 复制到 项目的3rd/libuv目录中 无libuv目录则创建

  • 将项目附件包含头文件目录 添加 3rd/libuv/include目录

  • 引入头文件

  • 编译不通过则 右击项目解决方案 -> 属性 -> C/C++ -> 常规 -> SDL检查 设为否
  • 链接lib库
  • 若还是编译不通过 则是头文件再包含一个 3rd/libuv/src 目录 因为有几个头文件在src目录 而C文件工作目录在src/win 则需要返回上一级

  • 因为考虑到跨平台 链接库文件还需要放在 链接器的命令行中 因为#pragma comment 是win才有的

  • so 右击 项目解决方案 -> 属性 -> 链接器 -> 命令行 添加 ws2_32.lib Psapi.lib Iphlpapi.lib Userenv.lib -> 应用

  • 则将原来的 pragma 删掉

  • 如果没加这几个库文件 会报一些链接错误 一般阅读一下错误提示去百度查一下就能发现需要链接哪个库了

另外 libuv/test 目录下是 官方写的测试代码 可以去阅读查看
