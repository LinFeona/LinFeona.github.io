title: Unity 打包Android环境配置

  • 资源准备 windows-Android-SDK 密码:1111 Java-jdk 密码:1111

java-jdk安装

  • 先查看 Unity 是否有jdk
  • Unity 打开首选项 edit -> preferences -> External Tools -> jdk 如果是勾选上的 并且是这样子的

  • 那么恭喜你 这一步可以跳过了 可往下拉到sdk的环节了

  • 1.安装jdk

  • 选择一个路径 最好默认的路径 记住这个路径 环境变量会用到

  • 直接下一步 默认下一步 直到完成安装

java环境配置

  • 复制jdk的路径

  • 右键我的电脑-属性-高级系统设置-环境变量-系统变量

  • 新建系统变量 变量名为 JAVA_HOME 变量值为 C:\Program Files\Java\jdk1.8.0_152 就是你jdk的路径

  • 找到系统变量的 PATH 双击它 新建一个变量值 %JAVA_HOME%/bin 若是没有 PATH 变量则新建

  • 新建系统变量 变量名为 CLASSPATH 变量值为 .;%JAVA_HOME%\lib;%JAVA_HOME%\lib\tools.jar; 前面还有一个· 后面还有一个分号

测试java环境

  • 按住 WIN + R 键 输入 cmd 回车

  • 在命令行窗口 输入 java -version 回车

  • 如果正常显示 这样 表示环境已经搭建成功了

sdk环境安装 要记住安装或解压的路径

  • 直接使用现成的 Windows-Android-sdk 在下载链接中的 android-sdk_r24.4.1-windows.zip 解压即用 非常的银杏
  • 或者 外力援助 使用Android Studio 在下载链接中的 Android-Studio.exe 安装非常的头疼

Unity 与SDK关联

  • 打开首选项 edit -> preferences
  • External Tools -> jdk 如果是如图3-1酱紫 那这步也跳过 否则那就 取消勾选 点击 Browse 选择jdk的位置

  • External Tools -> sdk 将勾选取消掉 点击 Browse 选择sdk的路径

打包设置

  • 检查一下你的场景是否加载到这里 如果没有的话 就点击 Add Open Scenes 将场景加载进来
  • 若你有多个场景 请依次打开 并且依次 Add Open Scenes 进来

  • 全部加载进来后再次打开你游戏的初始化场景 再点击 Add Open Scenes 保证它是打开游戏后运行的第一个场景

  • 再将平台切换到安卓

  • 打开 Player Settings 进行打包设置

  • 注意看包名格式 xxx.xxx.xxx 公司名.包名.游戏名

  • 最后 开始打包 build

  • 最后 打包错误锦集 点击外援

libuv session内存池的设计

可以阅读 更多文章!

session 原理

  • 每个socket 都会对应一个session跟客户端socket 所有的交流都在session中处理

  • session 会提供几个主流的接口
    发送数据 关闭连接 以及获取信息

  • 每当有一个用户进来的时候 就通过session.create 来创建一个session来对接新的用户

  • 每当有一个用户离开的时候 就通过 session.destroy 进行回收

  • session.create 以及 session.destroy 为从内存池中获取以及回收到内存池中

内存池设计原理

  • 一次性创建大量session
  • session.create时 获取一个
  • session.destroy 时 清空数据回收到池中
  • 分配器的管理

多个协议session 原理

  • 若有多个协议的连接 可能session中的数据结构会有所不同
  • session 可以采用继承模式 session基类则提供基础接口子类则区分这些不同的结构
    子类则区分这些不同的结构
  • 上层使用时只管使用session即可

Libuv c语言中的环境配置

可以阅读 更多文章!

简介

  • libuv是跨平台的异步事件库

libuv 下载

  • libuv官网
  • 下载直通车
  • 选择一个版本 下载 gz 后缀的源码

环境配置

  • 项目目录 src win32 3rd 目录 代码 运行时目录 第三方代码

  • 将 libuv/src 与 include 复制到 项目的3rd/libuv目录中 无libuv目录则创建

  • 将项目附件包含头文件目录 添加 3rd/libuv/include目录

  • 引入头文件

  • 编译不通过则 右击项目解决方案 -> 属性 -> C/C++ -> 常规 -> SDL检查 设为否
  • 链接lib库
  • 若还是编译不通过 则是头文件再包含一个 3rd/libuv/src 目录 因为有几个头文件在src目录 而C文件工作目录在src/win 则需要返回上一级

  • 因为考虑到跨平台 链接库文件还需要放在 链接器的命令行中 因为#pragma comment 是win才有的

  • so 右击 项目解决方案 -> 属性 -> 链接器 -> 命令行 添加 ws2_32.lib Psapi.lib Iphlpapi.lib Userenv.lib -> 应用

  • 则将原来的 pragma 删掉

  • 如果没加这几个库文件 会报一些链接错误 一般阅读一下错误提示去百度查一下就能发现需要链接哪个库了

另外 libuv/test 目录下是 官方写的测试代码 可以去阅读查看
